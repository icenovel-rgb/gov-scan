# 자격요건.md — 프로파일 양식 · 다중주체 판정

작업폴더에 두는 **창업자·기업 자격 프로파일**. 스킬 1단계에서 읽어 공고 신청자격과 대조한다.

## 핵심 개념 — 다중 신청주체 (OR 매칭)

한 사용자가 **여러 신청주체**를 가질 수 있다(예: 본인=기존 사업자, 제자=예비창업자, 공동창업).
공고마다 요구 자격이 다르므로, **주체 중 하나라도 자격을 충족하면 그 공고는 `해당`**으로 판정한다.

- 판정 시 각 주체별로 충족 여부를 따지고, **하나라도 충족 → `해당`**.
- `eligibilityReason`에 **어느 주체로 해당**인지 명시(예: "주체2(예비창업자)로 해당 — 예비창업패키지").
- 모든 주체가 명백히 배제 → `해당없음`. 정보 부족 → `검토필요`.

## 양식 (templates/자격요건.md)

`## 공통`(소재지·업종·관심분야·수혜이력) + `## 주체 N`(창업단계·사업자형태·대표자·인증·규모) 블록 반복.
자유서술도 허용하나, 아래 키가 채워질수록 정확하다.

## 스킬 발동 시 인터뷰 체크리스트

프로파일이 비었거나 모호하면 아래를 **주체별로** 확인하고, 사용자 답변으로 자격요건.md를 갱신한다.

### A. 창업 단계 (정부사업 1순위 필터) — 주체별
예비창업(사업자 전) / 창업 3년이내 / 7년이내 / 7년초과·무관 / 재창업. *기존 사업자는 설립일만 받으면 자동 분류.*

### B. 사업자·대표자 — 주체별
사업자 형태(개인/법인/없음) · 대표자 연령·성별(청년 만39세↓·여성 우대) · 특이자격(학생/교원/석박사/경력단절/장애). 동업이면 공동대표 각각.

### C. 규모·업종·지역 (공통)
소재지(시·도/시·군·구) · 업종(업태/종목·KSIC) · 연매출·상시근로자(소상공인/중소 판정) · 보유 인증·유형.

### D. 이력·우선순위 (공통)
정부지원 수혜 이력(중복수혜 제한) · 관심 분야 · 제외 키워드.

## 판정에서 자주 쓰는 기준

| 공고 제한 유형 | 프로파일에서 보는 값 |
|---|---|
| 창업단계("예비창업", "7년 이내") | 주체별 창업단계 / 설립일→업력 |
| 지역 제한("○○시 소재") | 공통 소재지 |
| 규모 제한("소상공인", "중소기업") | 기업유형 / 근로자수 / 매출 |
| 업종 제한("제조업", "지식서비스") | 업종 / KSIC |
| 대표자 요건("청년", "여성", "39세 이하") | 주체별 대표자 연령·성별 |
| 인증 요건("벤처기업 대상") | 기업유형·인증 |
| 중복수혜 제한 | 정부지원 수혜 이력 |

## 판정 산출(스킬이 채움)
- `해당`: 한 주체 이상이 명시 요건을 충족(또는 제한 없음). 사유에 해당 주체 명시.
- `해당없음`: 모든 주체가 명백히 배제.
- `검토필요`: 공고 요약만으로 판단 불가 → 상세URL 확인 필요. DB엔 `보류`로 기록.
